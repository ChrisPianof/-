## API

```tsx

<Status
  view='muted-alt'        // 'contrast' | 'muted' | 'muted-alt' | 'soft' (deprecated alias)
  color='green'           // 'green' | 'orange' | 'red' | 'blue' | 'grey' | 'purple'
  size={20}               // 20 | 24 | 32 | 40
  shape='rounded'         // 'rectangular' | 'rounded' — в Альфе ВСЕГДА 'rounded'
  uppercase={true}
  leftAddons={<DiamondsSIcon width={12} height={12} />}
```

**Дефолты:** `view='muted-alt'`, `color='green'`, `size=20`, `shape='rectangular'`, `uppercase=true`.

Дефолт `shape='rectangular'` не соответствует правилу Альфы — ставить `shape='rounded'` явно.

## Color — семантика 6 цветов

Цвет = функция семантики, не выбор по виду. Прогнать через decision tree (раздел «Алгоритм»).

| `color` | Семантика | Канонические лейблы |
|---|---|---|
| 🟢 `green` (Approve) | Полностью успешный финал | Одобрено · Исполнено · Готова · Открыт · Подключено · Подтверждено |
| 🟠 `orange` (Attention) | Частично успешный финал ИЛИ действие с вниманием | Одобрено частично · Исполнено частично · Требуются документы |
| 🔵 `blue` (Action) | Действие пользователя без алярма | На подпись · К оплате · К отправке |
| 🟣 `purple` (Process) | Процесс на стороне банка | В работе · Загружается |
| 🔴 `red` (Error/Risk) | Негативный финал · Ошибка · Критичный риск | Отклонено · Отменено · Не загружено · Не оплачено · Удалится сегодня · Просрочен · Заблокирована |
| ⚫ `grey` (Neutral) | Фоновое состояние без действий ИЛИ нейтральный финал | Черновик · Удалена · В архиве · Закрыт |

23 канонических лейбла. Whitelist — `Status` отрисовывает только это. Всё остальное — кандидат на исправление через UX-копи правила.

**Цвет `teal` не использовать** — поддерживается в коде core-components, но не входит в палитру статусной модели Альфы.

**`purple` рендерится через CSS-переменные `indigo`** (`--color-light-decorative-indigo` и т.д.) — naming-mismatch заложен в коде.

**Дисамбигуация «Отменено / Отклонено»** (оба 🔴): выбор по факту (кто отменил), детали — в Plate/Alert/деталке, не в самом Status. Примеры детализации: «Вы отменили заявку 6 мая», «Истёк срок получения карты».

## Алгоритм выбора цвета

```
Конечный статус процесса?
├─ ДА → Полностью успешный?
│       ├─ ДА → 🟢 ЗЕЛЁНЫЙ
│       └─ НЕТ → Частично успешный?
│                ├─ ДА → 🟠 РЫЖИЙ
│                └─ НЕТ → Негативный? → ДА → 🔴 КРАСНЫЙ
└─ НЕТ (середина или статус вне процесса)
        → Требуется действие пользователя?
          ├─ ДА → Повышенное внимание?
          │       ├─ ДА → 🟠 РЫЖИЙ
          │       └─ НЕТ → 🔵 СИНИЙ
          └─ НЕТ → Действие на стороне банка?
                  ├─ ДА → 🟣 ФИОЛЕТОВЫЙ
                  └─ НЕТ → Ошибка? → ДА → 🔴 КРАСНЫЙ
                           Риск?   → ДА → 🔴 КРАСНЫЙ
                           Нейтрально? → ДА → ⚫ СЕРЫЙ
```

**Динамический цвет.** «На подпись» может менять цвет 🔵 → 🟠 по приближению дедлайна. Реализация — на уровне продукта, Status сам не таймерит.

## View — зависит от фона страницы

| `view` | Фон страницы | Что даёт |
|---|---|---|
| `'contrast'` | Любой | Насыщенный фирменный цвет фона + белый текст |
| `'muted'` | БЕЛЫЙ `#ffffff` | Бледный цветной фон + цветной текст |
| `'muted-alt'` | СЕРЫЙ `#f2f3f5` и подобные | Более контрастный фон, тот же цветной текст |

`muted-alt` темнее `muted` (для серого фона нужен более контрастный chip). Дефолт пакета `muted-alt` — для случая, когда фон не указан явно.

`'soft'` ≡ `'muted-alt'` (один CSS-блок `.soft, .muted-alt`). Это алиас, deprecated. Codemod миграции:
```bash
npx @alfalab/core-components-codemod --transformers=status-soft src/**/*.tsx
```

## Uppercase

`uppercase=true` меняет САМ ШРИФТ-ТОКЕН (Medium → Bold + letter-spacing 1.25), а не только CSS `text-transform`. То есть включение uppercase = смена визуального веса и трекинга, не просто CAPS.

## Шрифт в проде

Alfa Interface Sans (с версии 5.0.0 от 04.02.2026). До этого использовался системный.

## UX-копи: 16 правил

```
1.  Дружелюбно, без «товарищ майор»
2.  Не двойные статусы
3.  Всегда «ё», без опечаток
4.  Согласуй с РОДОМ сущности (Заявка → Одобрена, Заявление → Готово, Терминал → Подключён)
5.  Несколько сущностей в продукте → СРЕДНИЙ РОД (Письмо/Платёж/Реестр → «Отправлено»)
6.  Сокращай — убирай имя сущности (Карта заблокирована → Заблокирована)
7.  Не подменяй сущность в цепочке
8.  Глагол в статусе ↔ глагол на кнопке (Закрыть → Закрыт)
9.  Status — короткий маркер; детали → Plate/Alert/иконка/отдельная колонка
10. Единые лейблы во всех каналах (IB ≡ Mobile App)
11. Не пиши о сущностях как о живых (Ожидает X → На X / К X / Не X)
12. ≤ 2 слов, простой язык
13. НЕ используй слово «Ошибка» (Ошибка загрузки → Не загружено)
14. Долгий процесс → ГЛАГОЛ (Загрузка → Загружается)
15. Финал → существительное / краткое причастие (Закрыт, Открыт, Подключён)
16. Без аббревиатур и терминов (ДДУ, Синхронизация)
```

## Что НЕЛЬЗЯ

- ❌ `view='soft'` в новом коде (deprecated, есть warning в dev). Использовать `'muted-alt'`.
- ❌ `color='teal'` в продуктах Альфы (не в палитре).
- ❌ Кастомные цвета через `className` override — нарушение DS Альфы. В core-components технически работает, но Web :: Core явно запрещает.
- ❌ `shape='rectangular'` — правило статусной модели «используем версию Rounded».
- ❌ Лейбл > 2 слов, аббревиатуры, технические термины («ДДУ», «КЭП», «OTP»).
- ❌ Слова «Ошибка X» в лейблах — заменять на «Не X».
- ❌ «Ожидает X» — заменять на «На X» / «К X» / «Не X».
- ❌ Имя сущности в лейбле, если ясно из контекста.
- ❌ Подменять сущность в цепочке статусов.
- ❌ Status как контейнер деталей.
- ❌ Произвольное название цвета по виду — цвет = функция семантики (decision tree).

## Limitations компонента

- Нет `Status.Group` / `StatusList` для последовательности статусов с разделителями.
- Нет встроенных анимаций перехода.
- Нет встроенной поддержки динамического цвета по таймеру (реализация — на уровне продукта).
- Нет responsive/desktop/mobile подпакетов — единый импорт `@alfalab/core-components/status`.
- Нет `disabled` / `loading` состояний.
- Не focusable, не реагирует на клик — это маркер, не control.

## Концепция (определения из артборда «Определения»)

**Статусная модель** — последовательность возможных состояний объекта и переходов между ними. Делает взаимодействие прозрачным, предсказуемым, помогает понять на каком этапе задача/документ и какие шаги доступны.

**Статус** — стадия жизненного цикла, отражает актуальность, степень завершённости и назначение. Помогает ориентироваться: использовать ли документ, дать ли обратную связь, или это финальная версия.

- Выбирать `view` по фону страницы (белый → `muted`, серый → `muted-alt`)
